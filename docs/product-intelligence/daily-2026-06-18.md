## Pattern Card 1 — Market Context Layer

1. Source  
   TradingView community layouts, smart-money-concept indicators, range/scanner suites.

2. What problem it solves  
   Traders must switch between charts, timeframes, and tools to form one market view.

3. What users like  
   One-screen clarity; fewer mental steps.

4. What is generic  
   "Dashboard" and "dashboard with tabs."

5. What is noisy  
   Repainting claims, signal spam, alerts without context.

6. What can inspire RangeClarity  
   A contextual frame around the chart instead of more indicators.

7. How to make it original  
   Position around "range clarity" not "signal confidence."

8. How it can improve the paid package  
   Add a structured context layer tied to range, location, and structure.

9. Beginner value score: 8  
   A single clear context layer reduces confusion.

10. Advanced value score: 7  
   Reduces workflow fragmentation for experienced traders.

11. Moat score: 6  
   Competition is high, so moat requires specific framing and integration depth.

12. Simplicity score: 7  
   Could clutter if not tightly designed.

## Pattern Card 2 — Tiered Setup Quality

1. Source  
   VCP, Stage analysis, and risk/reward frameworks in trading education.

2. What problem it solves  
   Beginners can’t tell cleaner setups from risky setups.

3. What users like  
   Concrete quality scores reduce judgment load.

4. What is generic  
   A, B, C grading of setups already exists in many forms.

5. What is noisy  
   Overcomplicated scoring with too many inputs.

6. What can inspire RangeClarity  
   A tier system that maps directly to range/position criteria.

7. How to make it original  
   Tie grades to measurable structural conditions, not narrative.

8. How it can improve the paid package  
   Tier quality becomes the clearest paid differentiation.

9. Beginner value score: 9  
   Provides simple decision structure.

10. Advanced value score: 6  
   Already familiar to experienced users, so novelty is lower.

11. Moat score: 7  
   If tightly specified, harder to copy than generic alerts.

12. Simplicity score: 8  
   Clean output, but input rules must stay simple.

2. What problem it solves

3. What users like

4. What is generic

5. What is noisy  
   Indicators added without a single controlling frame.

6. What can inspire RangeClarity

7. How to make it original  
   Integrate it into a coherent narrative bar, not just isolated numbers.

8. How it can improve the paid package

9. Beginner value score: 8

10. Advanced value score: 7  
   Useful for experienced traders as a sanity check.

11. Moat score: 7

## 🧭 Main Framing Question

How can RangeClarity become more than an indicator and feel like a complete market clarity system?

Guiding answer for today:  
RangeClarity should differentiate as a **context system**, not another chart indicator. That means the added value is not more formulas — it is better framing: range, location, risk context, and setup quality in one coherent view. Product intent, clean design, and precise terminology are the moat.
