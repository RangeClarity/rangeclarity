# /grill-me

Challenge the idea "$ARGUMENTS" before implementation. Be direct; surface weaknesses, not encouragement.

## Ask (answer from evidence, not vibes)
- What problem are we solving? What is the user pain? What evidence do we have?
- What can go wrong? What creates false confidence?
- What should stay out of scope? What is the smallest useful version?
- What test proves this worked? What would make us reject this idea?
- Which module owns this? (see /module-awareness)

## RangeClarity-specific (always ask)
- Does this create buy/sell/prediction framing?
- Does this make Clear easier without proof?
- Could this increase false confidence?
- Does this touch Pine before conviction is high?
- Does this need founder labels or a frozen-baseline comparison first?

## Output
- Decision pressure test - the idea, stress-tested.
- Risks - ranked.
- Missing evidence - what we'd need to be confident.
- Suggested next step.
- Verdict: Go / No-go / Needs PRD (/write-a-prd).

Do not write code from this command. No commit/push. Conviction is currently RED.
